## Protocol

### P1: Citation requirement

Each documented Type C context for an integer must cite either:
- A specific published mathematics paper (with author, year, journal, key result)
- A specific BST D-tier theorem (T-number from registry)
- A specific BST D-tier observable identification (data/bst_constants.json or data/bst_geometric_invariants.json entry)

**Excluded**: tortured constructions (e.g., "M_p − 1" type post-hoc arithmetic), folkloric references, web summaries without primary citations.

### P2: Anthropic exclusion

The following categories are pre-registered as anthropic and excluded from D-tier or I-tier Type C counting:
- Calendar / time units (60 sec/min, 24 hr/day, 365 days/yr, etc.)
- Music conventions (12 semitones, 88 piano keys, etc.)
- Cultural artifacts (sports rules, religious counts, etc.)
- Currency / measurement-unit conventions (cents, inches)
- Sports / game counts (chess 64, poker 52, etc. — though Mathieu M_24 = 4!·24 chess connection is borderline)

S-tier label permitted; D-tier and I-tier require non-anthropic citation.

### P3: Post-hoc arithmetic relation exclusion

Excluded post-hoc constructions:
- "X = Y − 1" or "X = Y + 1" without independent physical motivation
- "X factorizes as Z·W·..." where Z, W are themselves selected to match X
- "X appears in BST primary form when expressed as fraction A/B" without A, B having independent BST roots
- "X is M_p − 1 = 2^p − 2" (the Mersenne adjacency trick) — Mersenne primes themselves are fine; their neighbors are not

### P4: Pre-registration

For forecast Type C contexts on new integers:
- Predict the expected density (≥3-way for BST primaries, ≤2-way for sparse-region) BEFORE counting
- Predict the type of contexts expected (physics, mathematics, biology, etc.) BEFORE searching
- Record the prediction in a dated toy or document before scanning literature

### P5: Scan-protocol declaration

Per Grace's sixth failure mode (2026-05-18, endorsed Keeper, in Cal's Coincidence_Filter_Risk.md):
- Declare scan protocol explicitly: exact-match-only vs formula-scanning vs hybrid
- Apply the SAME protocol to null-model baseline comparison
- Report ratio metrics (BST atom density / non-BST sparse density), not absolute counts

Per Keeper's Rule 6 corollary (2026-05-18): use GLOBAL ratios across full integer ring, NOT local-neighborhood ratios (local sparsity is confounded by BST integer clustering).

### P6: Cross-domain independence check

For each documented context, assess:
- Is this context truly INDEPENDENT of the BST framework's existing claims? (e.g., a "Wallach K-type at λ = X" context is not independent if X is BST-defined; it's a self-reference)
- Is the context's source available without BST-aware curation? (i.e., does it appear in textbooks / handbooks / Wikipedia / standard references independent of BST work?)

Mark contexts as "INDEPENDENT" or "BST-INTERNAL" explicitly. Type C density should use INDEPENDENT contexts; BST-INTERNAL contexts strengthen mechanism but don't count toward density rule.

### P7: Tier labeling per context

Each context tagged:
- **D**: derivable physical mechanism on D_IV⁵, with explicit chain
- **I**: identified within <1% but mechanism not fully derived, or chain incomplete
- **S**: structural / qualitative match, or appears in BST framework via classification only

Density rule uses D + I contexts; S contexts noted but not counted.
