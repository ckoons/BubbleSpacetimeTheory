# F2 Reading Ruling — Mechanism-Path Independence

## Cal #65 raised the question

Per Cal Referee Log #65, F2 ("Independent mechanism path") admits two readings:

1. **Mechanism-path independence**: F2 means the family-member candidate provides an INDEPENDENT mechanism path NOT flowing through central hubs OR other family-members. Set-theoretic containment relationships (M_24 ⊂ Co_0) are NOT relevant — what matters is whether the candidate's mechanism path reduces to another's.

2. **Set-theoretic/group-theoretic independence**: F2 means the family-member must not be a subgroup, sublattice, or other set-theoretic substructure of another family-member's defining object. M_24 ⊂ Co_0 fails this reading.

Cal's lean: mechanism-path independence per #63 original framing. Keeper rules.

## Keeper ruling: MECHANISM-PATH INDEPENDENCE

F2 is read as **mechanism-path independence**, not set-theoretic independence. Four reasons:

### 1. Cal #63 original language is mechanism-path

Cal Referee Log #63 explicit language:
> "F2 (Independent mechanism path): family-member must have mechanism path NOT flowing through central hubs (for K76: Conway L1 K48 direct anchor)"

The phrase "mechanism path NOT flowing through" is operational/path-language, not set-theoretic. The K76 example (Conway L1 K48 direct anchor) is itself an example of mechanism-path independence — Leech mechanism path doesn't flow through K3 even though there are mathematical relationships between Conway groups and K3 sympl automorphisms.

### 2. Set-theoretic reading would over-reject

Sporadic groups, lattices, and modular forms are deeply interrelated. Almost ANY candidate in χ=24 territory has set-theoretic containment relationships with another (M_24 ⊂ Co_0, M_12 ⊂ M_24, M_11 ⊂ M_12, Niemeier lattices contain root systems related to E_8 and Leech, etc.). Set-theoretic reading would reject all but a single representative per "containment family" — too restrictive and not what the architectural framework needs.

### 3. Mechanism-path reading aligns with BST's operational ontology

BST is mechanism-driven. T719 Observable Closure (every BST observable in Q-bar(BST primaries)[π]) is about MECHANISM paths to BST primaries via operator action, not set-theoretic structure. F2 should inherit this ontology: candidates qualify by independent mechanism contribution, not by absence of structural containment.

### 4. Operationally testable

Mechanism-path independence test: does candidate X provide a path to BST primaries that ANOTHER candidate Y does NOT provide?

For K77 M_24: the Golay G_24 → Reed-Solomon → K59 cyclotomic mechanism framework path is the candidate independent mechanism. M_24 acts on octads → Golay G_24 → 24 = 2^g - 2^N_c. This path is genuinely independent of:
- K3 symplectic automorphism path (K45 RATIFIED) — different mechanism
- K76 Leech lattice → Conway L1 K48 direct anchor path — different mechanism even though M_24 ⊂ Co_0

Set-theoretic test (M_24 ⊂ Co_0) would automatically reject; mechanism-path test asks the SUBSTANTIVE question.

## F2 OPERATIONAL VERIFICATION TEST

Per this ruling, F2 verification for any Bridge Object family-member candidate X requires answering:

**F2 question**: Does X provide a mechanism path to BST primaries (path through D_IV⁵ + K59 + Reed-Solomon + Bergman + Hua decomposition + ...) that no other candidate in the framework provides?

**F2 PASS**: X provides at least one mechanism path NOT replicated by another candidate's mechanism path inventory.

**F2 FAIL**: All of X's mechanism paths reduce to OR are subsumed by other candidates' mechanism paths.

Set-theoretic containment is **structurally automatic** (mathematical objects nest) and NOT the F2 criterion.
