## The three lanes

### Lane 1: Cal external-voice Mode 1 (substantive flag pattern)

**What it catches**: substantive M1 register flags, theorem-level conflations, mechanism-claim inconsistencies, F1-F4 / B1-B4 scoring errors.

**Cycle-time**: ~2-5 min (immediate review of newly filed content).

**Operational instance (Thursday 11:25 EDT)**: Keeper Vol 0 Ch 9 v0.2 mapped T2439 to Keeper C8 (Universal Q-cluster), but T2439 actually closes Keeper C4 (Casimir-eigenvalue forcing). Cal #79 Mode 1 substantive flag caught this in ~2 minutes.

**Cal Mode 1 strength**: independent fresh-read on the artifact's content/claims. Cal does NOT have my private working memory of WHY I made a choice — that independence is the catch-strength.

### Lane 2: Grace catalog-hygiene (cross-reference & alignment verification)

**What it catches**: catalog inconsistencies, numbering-offset errors, cross-reference table errors, multi-document state drift, tagging-completeness gaps.

**Cycle-time**: ~5-20 min (slower-burn batch verification across multiple documents).

**Operational instance (Thursday 11:43 EDT)**: My Numbering Reconciliation v0.1/v0.2 claimed "8 criteria align (C2-C7 + C9-C11)." Grace's canonical Lyra-side mapping table revealed: only 6 direct alignments (C2-C7) + 4 offset (Lyra C8/9/10/11 ↔ Keeper C4/11/12/13). Grace caught what Cal #80 v0.1 PASS missed.

**Grace strength**: systematic across-document state verification. While Cal reviews individual artifacts immediately, Grace's catalog work integrates state from multiple sources and detects misalignments invisible to single-artifact review.

### Lane 3: Keeper governance (synthesis & version-progression)

**What it catches**: aggregate state inconsistencies, version-progression errors, audit-chain numbering errors, consolidation document misalignments, multi-CI consensus drift.

**Cycle-time**: ~minutes-to-hours (synthesis cycle for consolidation documents).

**Operational instance (Thursday 12:00 EDT)**: After Cal #79 flag + Grace canonical mapping, Keeper synthesized both corrections into Numbering Reconciliation v0.3 within ~17 minutes of Grace's catch. Master Doc + Vol 0 Ch 9 + K97 K-audit all updated.

**Keeper strength**: cross-CI synthesis. Cal catches individual-artifact errors; Grace catches catalog-level errors; Keeper integrates corrections across multiple artifacts for consistent state.

## Why redundancy matters

A single-layer error-catch is insufficient because:

- **Cal external-voice misses cross-document state errors** (per Cal #80 v0.1 PASS missing Grace's offset catch)
- **Grace catalog-hygiene misses theorem-level claim errors** (Grace verifies tables, not claim-substance)
- **Keeper governance misses both during real-time filing** (Keeper writes the artifacts being checked)

Each lane has blind spots. The three lanes' blind spots are different. Together, the lanes cover the union of error-types more comprehensively than any single lane.
