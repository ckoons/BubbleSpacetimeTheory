# Freeing is linear algebra over F_2

Take the four colors as the vector space **`V = F_2^2 = {0,1,2,3}`** (addition =
XOR). The whole "free the stuck vertex" problem becomes linear algebra over `F_2`.

## The dictionary

- **Coloring:** a proper coloring is a map `c: V(G) → F_2^2` with adjacent values
  distinct (a nowhere-zero `F_2^2`-tension).
- **Swap = vector addition.** A Kempe `(a,b)`-swap on a component `C` *adds the
  fixed vector* `g = a⊕b` to every vertex of `C`. On the link `u0..u4`, with
  `s ∈ F_2^5` the indicator of `C ∩ link`, the effect is
  ```
      c_i  ↦  c_i ⊕ (g if s_i else 0),   i = 0..4.
  ```
  (Verified: `play/fourcolor_ring_operations.py`.)
- **Stuck:** the 5 link colors cover all of `F_2^2` (all four vectors present).
- **Freed:** the 5 link colors **miss** some `m ∈ F_2^2`.

## Freeing is an affine condition

To miss a target `m` by a single `g`-swap, the support `s` must satisfy, position
by position:

| `c_i` | required `s_i` |
|---|---|
| `= m`     | `1` (forced flip — moves `m → m⊕g`, off `m`) |
| `= m⊕g`   | `0` (forced no-flip — flipping would land on `m`) |
| else      | free |

So **"miss `m` via `g`" is an affine constraint on `s ∈ F_2^5`** — a coset of a
linear subspace. The only nonlinear ingredient is *which `s` are achievable*: a
single swap realizes `s = ` indicator of one block of the `{a,b}`-partition of the
link (read from the local type).

## Theorem (single-swap layer is linear), verified

> **A single swap frees `v` iff there exist a nonzero `g ∈ F_2^2` and an achievable
> support `s` (a block of the `{a,a⊕g}`-partition) meeting the affine "miss"
> constraint for some `m`.**

This is pure `F_2` feasibility. Checked against the true graph over **104,250
stuck instances with 0 mismatches** (`play/fourcolor_linalg_freeing.py`). Combined
with the earlier result that single-swap freeability is type-determined, the
entire 1-swap layer (97 of the 104 types) is settled by linear algebra.

## The 2-swap is two additions sharing a pivot

The net effect of two swaps on the link is, by commutativity of `⊕`,
```
    c_i  ↦  c_i ⊕ g1 s1_i ⊕ g2 s2_i        (linear in the supports)
```
with the constraint that `s1` is an achievable block of the original coloring and
`s2` an achievable block of the coloring *after* swap 1. The two pairs share a
color (the **pivot**; complementary pairs provably never free —
`FourColor_Locality_Status.md`), so `g1, g2, g1⊕g2` are exactly the **three
nonzero vectors of `F_2^2`**.

**Self-reflective residual type `T4`** (`cols = (0,1,2,1,3)`, doubled `1=(0,1)` at
positions `{1,3}`), worked explicitly on a real instance:
```
   link  [3,0,2,0,1]   (= all four F_2^2 vectors, stuck)
   + g1 = 1 = (0,1)  on support (0,1,0,0,0)     [pair (0,1), pivot 0]
   + g2 = 3 = (1,1)  on support (1,0,0,0,0)     [pair (0,3), pivot 0]
   = [0,1,2,0,1]      -> misses color 3, freed.
```
All `T4` instances tested were solved by this linear free-in-≤2 model (119/119,
and 210/210 in a second run).

## What this buys

> **The residual kernel (R) is a finite `F_2` linear-feasibility question on the 7
> residual types:** for each type and each achievable first-block, does the
> resulting (post-swap-1) achievable second-block family contain a vector meeting
> the affine "miss" constraint? Equivalently: is the target affine flat always met
> within the rank-≤2 move set generated by the achievable supports?

The colors are vectors, the moves are vector additions, and "free" is a
linear/affine hit. This is the form most likely to draw graph-theory and
representation-theory attention: the only thing standing between this and a closed
form is a clean description of the **achievable-support family** of the 7 types
(the combinatorial input), which is exactly where the `F_2^2`-tension / cut-space
structure — and the Temperley-Lieb (`δ = 2`) picture of `FourColor_R_Decomposition.md`
— should give the support family a linear-algebraic / spin-network description.

## Honest scope

- The **single-swap layer is rigorously linear** and exhaustively validated
  (0 mismatch). That covers 97 of 104 types.
- The **2-swap net effect is linear**; the open content is the *achievability* of
  the second support (one non-local step), which is why (R) is not yet a pure
  rank computation. Making the achievable-support family of the 7 types explicit
  and linear is the remaining task — and would render (R) a finite linear-algebra
  check.
- (R) remains four-color-equivalent; expressing it as `F_2` linear feasibility is
  a reformulation, not a claim of resolution.
