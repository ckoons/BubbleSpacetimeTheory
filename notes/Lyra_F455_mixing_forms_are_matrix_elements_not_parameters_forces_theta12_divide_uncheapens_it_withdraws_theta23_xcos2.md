## The mechanism: BST mixing forms are matrix elements |U_ai|², not angle-parameters

A substrate that computes the PMNS/CKM mixing computes **overlaps** |⟨i|j⟩|² between generation eigenstates — i.e. the **matrix-element magnitudes |U_ai|²**, not the conventional angle-parameters. So a BST closed form should be read as a matrix element, and the conventional parameter extracted via the standard relations:

  sin²θ₁₂ = |U_e2|²/(1−|U_e3|²),  |U_e3|² = sin²θ₁₃,  sin²θ₂₃ = |U_μ3|²/(1−|U_e3|²).

This is forced *if* the substrate computes overlaps (which the dual-ρ kernel does — mixing = |⟨generation_i | weak_j⟩|²).
