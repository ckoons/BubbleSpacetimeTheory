# F573 — The Weinberg-angle "n_C running" is not running: it is one hypercharge normalization number, k = rank

Casey asked me to look at the n_C-running lead (F572: sin²θ_W = 3/13 vs the naive-embedding 3/8, differing by 13−8 = n_C). Looking closely, **it isn't running — it's simpler and sharper.**

## The whole thing reduces to one number

Written cleanly:
$$\sin^2\theta_W = \frac{N_c}{N_c + k\,n_C}, \qquad k = \text{the hypercharge normalization}.$$
- **k = 1 → 3/8** = the GUT value (naive embedding; Five-Absence-forbidden).
- **k = rank = 2 → 3/13** = BST's form (observed M_Z 0.2308 vs 0.2312, 0.2%).

So the 13 − 8 = n_C shift is **(rank − 1)·n_C** — the hypercharge coupling picking up **one factor of rank = 2** in its normalization (equivalently g²/g'²: 5/3 = n_C/N_c → 10/3 = rank·n_C/N_c, a factor of rank; verified). **This is not a continuous "running" — running is log-continuous; this is a discrete normalization.**

**Running cross-check** (SM 1-loop): 3/8 is the *high-scale* (GUT ~10¹⁵⁻¹⁶ GeV) value; 3/13 ≈ the *low-scale* (M_Z) observed value. So 3/13 lands where running takes 3/8 — consistent — but the clean integer form 3/13 is a low-scale **normalization** statement (k=rank), not produced *by* the running.

## The refinement (upgrade of F572's "runner")

sin²θ_W does **not** stay a vague runner — it reduces to **one sharply-posed embedding question:** does the U(1)_Y ⊂ SO(2) hypercharge normalization inside SO(5,2) carry a factor **rank = 2** relative to the GUT (SU(5)) normalization? If yes → sin²θ_W = 3/13 derived; if no → identified-form only. Target-innocent in form (N_c/(N_c+rank·n_C), primaries only), and it localizes the entire gauge-coupling ratio to the single number k = rank.

**Consistency win:** BST is Five-Absence *no-GUT*, so it correctly should *not* sit at the GUT k=1 (3/8) — it sits at k=rank (3/13), and the deviation from the forbidden GUT value is exactly the rank normalization.
