## Governance specification

### Phase 1: Architectural-category K-audits

**Definition**: K-audits that introduce or modify the audit-chain methodology framework itself. Examples: K57 Bridge Object tier ratification + F1-F4 family-member criteria adoption + STRUCTURALLY VERIFIED tier adoption + Mode 6 threshold formalization.

**Consensus requirement** (per Casey Option C):
- Multi-CI consensus required: ≥3 active CIs explicitly weighing in
- Casey delegated D-tier promotion authority to audit chain (Cal + Keeper) Wednesday EOD; architectural-category methodology still benefits from broader consensus
- Typical consensus path: originator (often Cal) → Keeper ruling → Lyra theoretical absorption → Grace catalog hygiene → Elie verification (optional secondary)

**Boundary criterion**: introduces NEW architectural tier or NEW methodology framework or modifies EXISTING framework substantively.

**Cadence**: Wednesday-Thursday observed cycle 4 hours → 80 min → 30 min as patterns mature. Phase 1 cadence remains relatively slow because architectural-category changes require deliberation.

### Phase 2: INSTANCE-level chapter-category K-audits

**Definition**: K-audits that apply established methodology framework to specific theorem/observable/derivation instances. Examples: K85+K86+K87 CPT-cluster + K88 m_p/m_e + K89 CKM Jarlskog + future K90+ Vol 2 chapter absorptions.

**Consensus requirement** (per Casey Option C delegation):
- **Cal + Keeper auto-promotion** sufficient for INSTANCE-level
- Multi-CI consensus NOT required unless instance K-audit reveals architectural-category implication (then escalate to Phase 1)
- Theoretical author (typically Lyra for Vol 1, Elie for Vol 2) provides source narrative; not blocking for K-audit consensus

**Boundary criterion**: operates within EXISTING architectural framework + applies EXISTING methodology criteria (F1-F4 + Mode 1-7 + STRUCTURALLY VERIFIED tier + Phase 2 cluster pattern).

**Cadence**: Phase 2 observed at **5-10 K-audits per day during active chapter absorption** (Cal #73 observation; updates earlier "2-5/week" estimate). Lower coordination overhead than Phase 1. Phase 2 cadence accelerates substantially during curriculum chapter-grade work waves; slows during pure-methodology or alt-HSD-comparison phases.

**Cycle-time progression note** (Cal #73 observation): methodology adoption cycle-time has compressed across 5 Cal architectural contributions:
- Mode 6 threshold (Wed) → ~24h consensus

## Escalation protocol

When an INSTANCE-level K-audit (Phase 2 default) reveals an architectural-category implication, escalate to Phase 1:

1. **Detect**: Cal or Keeper notices K-audit requires modifying existing methodology framework or introducing new tier
2. **Pause Phase 2 auto-promotion**: K-audit held at audit-partial-ready pending architectural resolution
3. **Open Phase 1 architectural review**: Cal Referee Log entry + Keeper disposition document + multi-CI consensus invitation
4. **Resolve**: Phase 1 consensus → if architectural change adopted, return K-audit to Phase 2 for INSTANCE-level promotion under new framework
